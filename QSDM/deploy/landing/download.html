QSDM Hive

Install Hive on Windows or Linux for CELL wallet management, signed QSDM tasks, integrations, NVIDIA-attested mining access, and pooled CPU, GPU, and RAM participation.

Windows installer

Recommended for all users. Hive is the public release path for wallets, tasks, integrations, and NVIDIA mining access. Hive bundles the console miner it supervises; no separate consumer GUI miner is shipped.

Version 1.3.90

Linux x86-64

Use the AppImage on a desktop Linux distribution with GTK 3. Linux Hive reads the ledger directly from canonical QSDM Core while task metadata uses the restricted home-validator gateway. Ordinary users do not need to install or run a local Core. Wallet keys remain local, the native QSDM signer and console miner are bundled with Hive, wallet-address copying uses the native desktop clipboard, and task eligibility reads the active signer's confirmed CELL balance. Legacy native task labels are normalized to CELL, so a funded signer can start eligible gateway tasks normally. Signed miner enrollment is submitted to canonical Core. Transient route failures now show Reconnecting, retain confirmed values, and require repeated failures before a short outage circuit opens. Actual chain mismatches still fail closed.

Edge Control 1.3.3

Agent and Relay utilities

Hive already includes the correct agent and NVIDIA helper for its operating system. The standalone bundles add a local Edge Control window for trusted laboratory or office computers. These are support utilities, not another QSDM client. Choose Agent or Relay, paste one pairing code, and set CPU, GPU, or RAM limits with sliders. QSDM Hive remains the only desktop client and assumes the Mother Hive role for the group.

Mining paths

NVIDIA-attested protocol mining

QSDM Miner task

Requires an eligible NVIDIA Turing-or-newer GPU for v2 identity, attestation, and CUDA SHA3 proof search. The Tensor-Core fork remains a separate future consensus activation. Consumers run this from Hive; advanced operators can use qsdmminer-console.

Miner requirements
CPU, GPU, and RAM

Pooled edge participation

Users without NVIDIA GPUs can contribute bounded CPU or RAM. Supported NVIDIA systems can contribute CUDA capacity separately from protocol mining.
